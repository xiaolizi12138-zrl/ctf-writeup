1. 题目信息
题目名称：ctf.show_web5

题目类型：Web / PHP 代码审计

考察知识点：PHP 弱类型比较（==）、MD5 绕过（0e 科学计数法）

2. 源码分析
访问题目页面，看到一段 PHP 代码：

php
<?php
error_reporting(0);
$flag="";
$v1=$_GET['v1'];
$v2=$_GET['v2'];
if(isset($v1) && isset($v2)){
    if(!ctype_alpha($v1)){
        die("v1 error");
    }
    if(!is_numeric($v2)){
        die("v2 error");
    }
    if(md5($v1)==md5($v2)){
        echo $flag;
    }
}else{
    echo "where is flag?";
}
?>
逻辑分析：

输入限制：通过 GET 方式接收两个参数 v1 和 v2。

v1 限制：ctype_alpha($v1) 要求 v1 的值必须全是字母。

v2 限制：is_numeric($v2) 要求 v2 的值必须全是数字。

最终判断：md5($v1) == md5($v2)，即两个参数的 MD5 值必须相等。

3. 漏洞原理
这里使用了 PHP 的 ==（松散比较/弱比较），而不是 ===（全等比较）。

在 PHP 中，如果一个字符串以 0e 开头，当它参与 == 比较时，PHP 会将其解析为科学计数法。

例如：0e12345 会被计算为 0 * 10^12345，结果是 0。

因此，只要找到两个不同的字符串，它们的 MD5 值都以 0e 开头，在 PHP 中使用 == 比较时，它们都会被解析为 0，从而导致 0 == 0 成立，绕过判断直接输出 Flag。

4. 解题过程
题目要求 v1 必须全是字母，v2 必须是数字。我们需要找两个 MD5 值是 0e 开头的字符串，分别给 v1 和 v2 使用。

为 v1 准备字符串（纯字母）：
在 PHP 圈有一个非常著名的字符串 QNKCDZO，它的 MD5 值正好是 0e 开头。

md5('QNKCDZO') = 0e830400451993494058024219903391

为 v2 准备字符串（纯数字）：
找到一个纯数字 240610708，它的 MD5 值也是 0e 开头。

md5('240610708') = 0e462097431906509019562988736854

构造 Payload：
在浏览器地址栏中访问：

text
?v1=QNKCDZO&v2=240610708
验证结果：

ctype_alpha('QNKCDZO') 为真，通过。

is_numeric('240610708') 为真，通过。

md5('QNKCDZO') == md5('240610708') 为真，通过。

页面成功输出 flag。
